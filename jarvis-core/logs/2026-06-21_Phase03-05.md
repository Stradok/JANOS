## Bugs Fixed

### 1. `agents/crew_agents.py` — CriticAgent.validate() never returned score
**Root cause:** Prompt asked for `SCORE: -1.0 to 1.0` but response was parsed to only `{"raw": ..., "valid": bool}`. Every call to `val.get("score", 0.0)` returned 0.0.  
**Fix:** Parse the `SCORE:` line from the LLM response. Default to 0.7 (optimistic) when LLM doesn't return a parseable score. Clamp to [-1.0, 1.0].  
**Verified:** `CriticAgent.validate()` with mocked LLM returning `SCORE: 0.85` → `{"score": 0.85, "valid": True}`. Invalid response with `SCORE: -0.5` → `{"score": -0.5, "valid": False}`.

### 2. `core/routing.py` — record_model_result() broken average formula
**Root cause:** `avg = (score + (sc + fc)) / max(1, sc + fc)` — same bug as `record_agent_result()` had in Phase 2, but in the twin function. `avg_score` was also not read from SELECT, so running average was impossible.  
**Fix:** Added `avg_score` to SELECT. Changed to proper running average: `avg = (old_avg * total_old + score) / total_new`.  
**Verified:** 2 calls (0.9, -0.1) → avg=0.400, matches expected (0.9-0.1)/2.

### 3. `core/reasoning_pipeline.py` — outcome_score never set in happy path
**Root cause:** `ctx.outcome_score = 0.0` was only in the `else: (no critic)` branch. When critic ran, score was validated and errors appended but `ctx.outcome_score` was never written. Combined with Fix #1, outcome_score was now set from `validation.get("score", 0.7)`.  
**Fix:** Added `ctx.outcome_score = validation.get("score", 0.7)` after critic validates. Default 0.0→0.7 when no critic.

### 4. `core/strategy_refiner.py` — 24h wait before first refinement
**Root cause:** `_loop()` slept before running, so first refinement cycle couldn't be observed or tested.  
**Fix:** Added `run_immediately: bool = False` parameter to `start()`. When True, calls `run_refinement()` before creating the background task. In `main.py`, changed `await refiner.start()` → `await refiner.start(run_immediately=True)`.

### 5. `modules/feedback_module.py` — user ratings not bridged to ScoringEngine
**Root cause:** `_update_skill_score()` only called `learning_engine.process()` — the old v2 learning engine. `core/scoring.py` (Phase 5 ScoringEngine) was never called from feedback path.  
**Fix:** Added ScoringEngine import and call inside `_update_skill_score()`. Wrapped in try/except. Now every 1-5 user rating also records `record_action(agent, success=(rating>=4))`.

## Features Added

### Phase 3 — Self-Healing Retry Loop
`/api/v2/chat` now retries when `CriticAgent` rejects output:
```python
while not val.get("valid", True) and ctx.retry_count < ctx.max_retries:
    ctx.retry_count += 1
    # Re-execute with critic feedback injected into prompt
    retry_prompt = f"{ctx.task_plan}\n\nPrevious attempt was rejected:\n{feedback}\n\nFix the issues and retry."
    ctx.agent_output = await agents["executor"].execute_step(retry_prompt)
    val = await agents["critic"].validate(ctx.agent_output, req.input, ctx.errors)
```
Max retries = `ctx.max_retries` (default 3). Response includes `"retries": N`.

### Phase 3 — RoutingEngine wired to /api/v2/chat
After every chat, records agent and model outcomes:
```python
routing_eng.record_agent_result(ctx.selected_agent, success=..., score=...)
routing_eng.record_model_result(ctx.selected_model, ctx.task_type, success=..., score=...)
```
`/api/v3/rankings` now returns real data after real tasks.

### Phase 5 — ScoringEngine wired to /api/v2/chat
After every chat, calls `ScoringEngine().record_action(agent, success=...)`.  
`/api/v5/scores` endpoint now returns meaningful data.

### Phase 5 — /api/v5/scores endpoint
New endpoint exposes all ScoringEngine utility scores:
```
GET /api/v5/scores → {"scores": [{"action": "executor", "score": 1.5, ...}]}
```

### CLI — chat.py
New `chat.py` connects to the running server and supports:
- `--full` for Phase 2-5 full pipeline (`/api/v2/chat` with RAG, scoring, retries)
- `--v1` for legacy orchestrator
- `!<cmd>` for Phase 4 command interface
- `/scores` shortcut showing agent/model rankings
- Shows score, retries, episode ID per response
