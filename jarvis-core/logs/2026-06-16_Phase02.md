# Phase 2 — Full LangGraph + Agent Layer — 2026-06-16

## Summary
Replaced the MVP state machine with a full LangGraph-style state machine with conditional edges for retry/escalate/fail. Built 6 CrewAI-style agent roles (Planner, Executor, Critic, Searcher, MemoryAgent, FileOperator) and an AutoGen debate system for complex multi-step reasoning.

## What Was Built

### 1. Full LangGraph State Machine (`core/langgraph_machine.py`)
- `LangGraphMachine` class — graph-based state machine with add_node, add_conditional_edges, set_entry_point
- `WorkflowContext` dataclass with `failed` property
- 13 states: `INPUT → RAG_RECALL → MODEL_SELECT → TASK_DECOMPOSE → AGENT_ALLOCATE → EXECUTE → CRITIC_VALIDATE → STORE_EPISODE → RESPOND → END` + `RETRY`, `ESCALATE`, `FAIL`
- `RETRY` state auto-increments retry_count and loops back to TASK_DECOMPOSE (max 3 retries)
- `ESCALATE` state collects all errors and reasoning steps for user handoff
- `FAIL` state produces structured error response

### 2. CrewAI-Style Agents (`agents/crew_agents.py`)
- `AgentRole` base class with `think()` method routing through our own LLMClient
- 6 agent roles:
  - **PlannerAgent** — Task decomposition specialist. `create_plan()` with RAG context
  - **ExecutorAgent** — Execution specialist. `execute_step()` with tool awareness
  - **CriticAgent** — QA + error analysis. `validate()` with structured scoring, `diagnose_failure()` with past failure context
  - **SearcherAgent** — Information retrieval. `search_memory()` + `synthesize()`
  - **MemoryAgent** — Memory management. `store()`, `recall_relevant()`
  - **FileOperatorAgent** — File system. `read_file()`, `write_file()` with backup support
- `AgentRegistry` — global agent registry matching CrewAI pattern
- `initialize_agents()` — factory function wiring all agents with LLM + memory

### 3. AutoGen Debate (`agents/autogen_debate.py`)
- `AutoGenDebate` class — lightweight multi-agent debate with up to 3 rounds
- Each round: Planner proposes → Executor critiques → Critic evaluates/scored
- Consensus detection when critic outputs `FINAL_PLAN:`
- Routes through our own LLMClient (no OpenAI dependency)
