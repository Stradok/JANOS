## Rationale

Previously, running JANOS required remembering multiple commands scattered across `setup.sh`, `demo.py`, `chat.py`, README, and the Phase logs. No single place existed to:
- Start the server in dev vs prod mode
- Run the test suite
- Check health/rankings
- Init databases
- Pull Ollama models

The Makefile consolidates all of this under `make <target>`.

**Key design decisions:**
- `PYTHONPATH=.` on all Python invocations so `core/`, `agents/`, `tools/` resolve correctly
- `_venv-check` guard on every target that needs `venv/bin/` â€” prints clear instructions if missing
- `PORT=8000` and `HOST=0.0.0.0` overridable: `make dev PORT=9000`
- `MODEL=` and `CAP=` parameters for Ollama pull and tool generation targets
- `db-reset` has a typed confirmation prompt (`yes`) to prevent accidental data loss
- Complex Python logic extracted to `scripts/` â€” not embedded in Makefile (avoids TAB/indentation ambiguities)
- OS detection (`uname`) for `make open` (xdg-open on Linux, open on macOS)
