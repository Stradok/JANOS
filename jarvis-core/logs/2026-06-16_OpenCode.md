# OpenCode Refactoring Log — 2026-06-16

## Summary
Completed a comprehensive architectural refactoring of JANOS from a Windows-bound monolithic assistant into a Linux-native, production-ready multi-agent framework with persistent memory, scoring engine, and pluggable tools.

## What Was Done

### 1. New Architecture (`core/`, `agents/`, `tools/`)
Created a clean modular structure alongside the existing codebase:

| Layer | Files | Purpose |
|-------|-------|---------|
| `core/` | `config.py`, `llm.py`, `memory.py`, `scoring.py`, `logger.py` | Backbone: config singleton, unified Ollama interface, short+long term memory, utility scoring engine, structured JSON logging |
| `agents/` | `base.py`, `registry.py`, `planner.py`, `executor.py`, `researcher.py`, `reflector.py`, `orchestrator.py` | Multi-agent framework: think→act→observe loop with specialized agents and a top-level dispatcher |
| `tools/` | `base.py`, `registry.py`, `math_tool.py`, `time_tool.py`, `weather_tool.py`, `web_search_tool.py`, `file_tool.py`, `system_tool.py`, `browser_tool.py`, `media_tool.py` | Pluggable tool interface — each tool is independently testable with a consistent `execute(**params)` contract |

### 2. Windows → Linux Migration
Fixed **17 files** containing Windows-specific code:

| File | Fixes Applied |
|------|-------------|
| `app_launcher_module.py` | Replaced hardcoded `C:\` paths with Linux `shutil.which()` lookups; `pygetwindow` → `wmctrl`/`xdotool`; `os.startfile` → `xdg-open`; removed `powershell.exe`, `notepad.exe`, `calc.exe`, `explorer.exe` references |
| `system_control_module.py` | `rundll32 LockWorkStation` → `loginctl lock-session`; `shutdown /s` → `systemctl poweroff`; `shutdown /r` → `systemctl reboot`; `rundll32 SetSuspendState` → `systemctl suspend`; `os.startfile` → `webbrowser.open`; pycaw → pulsectl with pycaw fallback |
| `screen_reader.py` | Hardcoded `C:\Program Files\Tesseract-OCR\tesseract.exe` → `shutil.which("tesseract")` |
| `smart_tts_module.py` | Removed PowerShell Media Player COM fallback; replaced with `ffplay`/`paplay`/`aplay`/`mpg123` chain; `os.startfile` → `xdg-open` |
| `spotify_module.py` | Removed `shell:AppsFolder`, `APPDATA`, `Spotify.exe`, `pygetwindow`; added `spotify` binary, flatpak, snap, and `xdg-open` spotify: protocol |
| `youtube_module.py` | `os.startfile` → `webbrowser.open`; User-Agent `Windows NT 10.0` → `X11; Linux` |
| `browser_module.py` | `os.startfile` → `webbrowser.open`; User-Agent `Windows NT 10.0` → `X11; Linux` |
| `daemon_module.py` | Removed `os.name != "nt"` conditional; unified `disk_usage("/")` |
| `orchestrator_module.py` | Prompt: "Windows PC" → "Linux PC"; example path `C:\Users` → `/home` |
| `system_agent.py` | "Windows PC" → "Linux PC" |

## User Remarks (Post-Phase 1-5)

### GPU & CPU Utilization
The GPU usage is not optimized — the cores are being used unevenly in the CPU also.

### Missing Terminal Access
It still does not have access to terminal. (The privilege shell exists via `/api/v4/exec` with `--confirm` but no interactive terminal session.)

### No Screen Awareness
It cannot see what is on the screen. (No screen capture / OCR / vision pipeline integrated into the new engine.)

### Reasoning Limitations
It still lacks basic common sense and works highly on steps — the planner decomposes everything into rigid sequential steps rather than exercising genuine understanding.

### Future Direction: Soul & Mind
It needs a soul and a mind (a creative one) — maybe it should learn while talking to a real LLM like Claude, etc. Continuous learning from a conversational teacher model rather than just scoring past actions.
