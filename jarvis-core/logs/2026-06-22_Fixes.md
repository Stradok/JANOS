## Issues Fixed

### 1. Spotify — was opening browser instead of the desktop app

**Root cause:** `SpotifyModule._search_and_play()` required `pyautogui` for keyboard injection. `pyautogui` is not installed (raises `DisplayConnectionError` at import → `PYAUTOGUI_AVAILABLE = False`). The media agent LLM saw the error, fell back to the `browser` tool, and opened `open.spotify.com` — where the user has no account.

**Fix — `modules/spotify_module.py` — full rewrite:**
- Added priority control chain (tries each in order until one succeeds):
  1. **`playerctl`** — MPRIS2 via CLI (cleanest, if installed: `sudo apt install playerctl`)
  2. **`dbus-send`** — D-Bus MPRIS2 for play/pause/next/previous — zero extra install, built into every D-Bus desktop. Works because Spotify registers `org.mpris.MediaPlayer2.spotify` when running.
  3. **`xdotool`** — keyboard injection into the app window (if installed: `sudo apt install xdotool`). Uses `Ctrl+K` to open search bar, types query, presses Enter.
  4. **`xdg-open spotify:search:QUERY`** — Spotify URI scheme registered on this system (`spotify_spotify.desktop`). Opens the Spotify app directly at the search results for the query. Always available.
- Added `_now_playing()` — reads track metadata via playerctl or D-Bus MPRIS2 metadata property.
- Added `_dbus_metadata()` — parses `xesam:title` / `xesam:artist` from `dbus-send` output.
- Confirmed: Spotify is installed via snap at `/snap/bin/spotify`. `wmctrl` available for window focus. `dbus-send` available for MPRIS2.

**Fix — `modules/agents/media_agent.py` — updated system prompt:**
- Reduced `max_steps` from 12 → 10.
- Removed `browser` from the media agent's tool list entirely (in `__init__.py`).
- Added explicit rule: **NEVER use `browser` tool for Spotify — the desktop app is installed.**
- Step-by-step Spotify fallback sequence documented in the prompt.
- Clarified YouTube-only use case for `youtube` tool.

**Fix — `modules/__init__.py` — tool map:**
- `"media"` tool list: removed `"browser"`, kept `["spotify", "youtube", "keyboard_mouse", "screen_reader", "app_launcher"]`.

### 2. Email — looping on email.com, never composing

**Root cause:** The communication agent used `browser.open("https://mail.google.com")` which launches a **fresh Playwright Chromium instance** with no stored session. Gmail shows the login page. The agent sees an unfamiliar page, doesn't know how to log in, and retries opening Gmail → infinite loop. The URL it generated sometimes degraded to `email.com` as the LLM hallucinated a simpler URL.

**Fix — `modules/agents/communication_agent.py` — new email strategy:**
- **Primary method:** Build a Gmail compose URL with all fields pre-filled:
  ```
  https://mail.google.com/mail/?view=cm&fs=1&to=EMAIL&su=SUBJECT&body=BODY
  ```
  Opening this URL skips the Compose button entirely — Gmail opens directly in compose mode with To/Subject/Body already filled in.
- Open the URL via `system_control.open_url` (calls `webbrowser.open()`) — this opens in the **user's real Chrome/Firefox where they are already logged in to Gmail**. Not a fresh Playwright browser.
- Wait 2 seconds, then `keyboard_mouse.hotkey(["ctrl", "enter"])` to send.
- **Fallback:** `mailto:EMAIL?subject=SUBJECT&body=BODY` via `system_control.open_url` — opens the system's default mail client.
- Added explicit rules: NEVER open `email.com`. NEVER retry the same URL more than once. NEVER use `browser` tool for Gmail (Playwright has no session).

**Fix — `modules/__init__.py` — tool map:**
- `"communication"` tool list: added `"system_control"` as first tool, changed order to `["system_control", "keyboard_mouse", "screen_reader", "browser", "smart_tts"]`.
- `browser` kept as last resort for WhatsApp Web / Discord (those sites don't require a saved session for basic interaction).

### 3. Terminal / system access — JAN didn't know about the system

**Root cause:** The system agent had `system_control` in its tool list but the system prompt showed no examples of `run_shell`. The LLM didn't know it could execute arbitrary shell commands. The agent also lacked `file_manager` (which has its own `run` action).

**Fix — `modules/agents/system_agent.py` — full prompt rewrite:**
- Increased `max_steps` from 10 → 15.
- Added full `system_control.run_shell` examples:
  - `uname -a && lsb_release -a` — OS info
  - `which spotify || snap list | grep -i spotify` — check if app is installed
  - `ps aux --sort=-%cpu | head -15` — top processes
  - `pkill -f chrome` — kill a process
  - `free -h && df -h /` — memory/disk usage
  - `nvidia-smi 2>/dev/null` — GPU usage
  - `sudo apt install -y APP` — install software
- Added `file_manager` examples (`list`, `read`, `run`).
- Added **Self-discovery rules**: when unsure if something is installed, run `which APPNAME` first, don't guess.
- Added **Self-learning rules**: on failure, run shell to discover the right command, check `man COMMAND`, try alternatives before giving up.

**Fix — `modules/__init__.py` — tool map:**
- `"system"` tool list: added `"file_manager"` → `["app_launcher", "system_control", "file_manager", "keyboard_mouse", "screen_reader"]`.
- `"self_improvement"` tool list: added `"system_control"`, `"web_search"`.
- `"automation"` tool list: added `"system_control"`, `"file_manager"`.
- `"coding"` tool list: added `"system_control"`.

### 4. Startup system probe — JAN now knows what's installed

**Fix — `modules/__init__.py` — added `_probe_system()` at module load:**

- Scans for: `spotify`, `playerctl`, `xdotool`, `wmctrl`, `xdg-open`, `ffmpeg`, `mpv`, `vlc`.
