## Bugs Fixed

### 1. `dual_llm_module.py` — Dead code after `return`
Instance attributes (`big_model_loaded`, `last_big_model_use`, `idle_timeout`, `stats`, `_lock`, `_idle_timer`) were placed after a `return []` statement inside `_fetch_available_models()`. They were unreachable — the module would crash the first time any method tried to access them.
**Fix:** Moved all six attributes into `__init__()`.

### 2. `main.py` — Missing `app` object + double startup handler
- `app = FastAPI(...)` was lost during refactoring, causing `app.mount(...)` and all `@app.get/post` decorators to throw `NameError` at import time.
- Two separate `@app.on_event("startup")` handlers (`on_startup` + `cache_models`) — deprecated in FastAPI 0.137.1.
**Fix:** Converted both to a single `@asynccontextmanager lifespan(app)` function. Restructured file so `app = FastAPI(title="...", lifespan=lifespan)` is created after the lifespan definition. Merged `cache_models` logic into lifespan startup block.

### 3. `requirements.txt` — Missing runtime packages
Three packages were actively used in modules but not listed, causing silent import failures:
- `openai-whisper` — STT (`smart_tts_module.py`, `screen_reader.py`)
- `edge-tts` — TTS (`smart_tts_module.py`)
- `pytesseract` — OCR (`screen_reader.py`)

All three were installed manually (`venv/bin/pip install`) and added to `requirements.txt`.
**Fix:** Complete rewrite of `requirements.txt` — organized by category, all core deps pinned, optional features documented in comments.

## New Modules Added

### `modules/autonomous_recovery.py` (~595 lines)
Self-healing engine. Intercepts any module failure, classifies the error, and tries a prioritized stack of fixes before escalating to the user.

**Error types classified:** `import_error`, `not_installed`, `cmd_not_found`, `display_error`, `ollama_offline`, `connection_error`, `timeout`, `unknown_action`, `attribute_error`, `permission_error`

**Recovery stack (in order):**
1. Solution cache — instant replay of previously solved errors (SQLite)
2. `pip install` missing Python package
3. `apt install` missing system package
4. Set `DISPLAY` env variable (headless/X11 fix)
5. Start Ollama if offline
6. Map unknown action alias → real module.action
7. Retry after transient failure
8. Web research via `web_search` module
9. Generate new module capability via `module_generator`
10. Structured diagnosis escalation to user

**API actions:** `stats`, `log`, `cache`
**DB tables:** `solution_cache`, `recovery_log` (in `memory/jarvis_memory.db`)

### `modules/feedback_module.py` (~246 lines)
Closes the self-improvement loop. After every task, registers it as pending feedback. User rates 1-5 via `/feedback` API. Rating → `outcome_score` → learning engine skill memory.

**Rating scale:** 1=−1.0, 2=−0.5, 3=0.0, 4=+0.5, 5=+1.0
**API actions:** `collect`, `register`, `stats`, `worst`, `pending`, `history`
**DB tables:** `task_feedback`, `pending_feedback`
