# Phase 4 — Command Interface + Privilege System — 2026-06-16

## Summary
Built a `/command` interface for structured agent invocation plus a sudo-aware privilege escalation system with full audit logging. Both the command interface and self-description system are dynamic — they read from live registries at runtime.

## What Was Built

### 1. Command Interface (`core/commands.py`)
- `parse_command(input_str)` — parses `/command args` from user input, uses `shlex.split` for proper quoting
- `CommandHandler` — registry of commands with descriptions and usage
  - `register(name, handler, description, usage)` — add commands
  - `execute(command, args, context)` — dispatch to handler
  - `list_commands()` — return all registered command metadata
  - `help_text()` — formatted help output
- `SystemDescriptor` — self-description system, answers "what can you do?"
  - `describe()` — dynamic readout of agents, tools, models, commands, hardware
  - Updated from live registries on every call

### 2. Privilege Escalation (`tools/privileged_shell.py`)
- `PrivilegedShell` — sudo-aware shell execution:
  - `execute()` — runs commands with timeout, safety checks, full logging
  - Destructive command detection (rm -rf, dd, mkfs, format, etc.)
  - `--confirm=True` required for destructive operations
  - Automatic `.bak` backup support via `file_backup()`
  - Undo log at `memory/undo_log.jsonl`
  - Audit log at `memory/privilege_audit.jsonl`
  - Every action logged with timestamp, command, cwd, sudo flag, justification, exit code

## Endpoints Added
| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/api/v4/command` | POST | Execute `/command` (parse → dispatch → respond) |
| `/api/v4/exec` | POST | Execute privileged shell command (with confirm/justification) |
| `/api/v4/audit` | GET | Get privilege audit log entries |
