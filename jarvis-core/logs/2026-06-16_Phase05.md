# Phase 5 — Continuous Learning + Strategy Refinement — 2026-06-16

## Summary
Built the continuous learning system that runs as a background job, analyzes episodic memory for low-scoring patterns, adjusts agent/model weights based on success rates, compresses old episodes, and can generate new tool module stubs for missing capabilities.

## What Was Built

### 1. Strategy Refiner (`core/strategy_refiner.py`)
- `StrategyRefiner` — background job running at configurable interval (default 24h):
  - `refresh_models()` — re-poll `ollama list` for new/dropped models
  - `run_refinement()` — full refinement cycle:
    - Updates model cache
    - Analyzes episodic memory stats (total episodes, avg score)
    - Detects failure patterns per task type
    - Compacts old episodes when count > 100 (30 day cutoff, top 200 kept)
    - Reads agent/model rankings from routing engine
  - `generate_module_stub(capability)` — uses LLM to generate a new Python tool module:
    - Takes capability description
    - Generates code following BaseTool pattern
    - Strips markdown code fences from LLM output
    - Returns ready-to-save code

### 2. Startup Integration
- Strategy refiner initialized at startup with access to memory, router, routing engine, and LLM
- Shutdown handler stops the refiner gracefully
- Manual trigger via API endpoint

### 3. Tool Registry v2 (`tools/registry_v2.py`)
- `ToolRegistryPhase2` — extended registry with `describe()` method
- Supports registered Phase 2+ tools alongside existing tools

## Endpoints Added
| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/api/v5/refine` | POST | Trigger strategy refinement cycle (refresh models, analyze memory, compact, read rankings) |
| `/api/v5/generate-tool` | POST | Generate a new tool module stub for a missing capability via LLM |
