### Bugfix: pyautogui `DisplayConnectionError` crashes module loading
- **Files**: `modules/system_control_module.py`, `modules/keyboard_mouse_module.py`, `modules/spotify_module.py`, `modules/screen_reader.py`
- After installing pyautogui via pip, `mouseinfo` dependency calls `Display(os.environ['DISPLAY'])` at import time
- On missing/broken X display, raises `DisplayConnectionError` (not `ImportError`)
- Changed `except ImportError:` to `except Exception:` on all module-level pyautogui imports

### New: pip_install and run_shell actions
- **File**: `modules/system_control_module.py`
- Added `_pip_install(package)` — runs `sys.executable -m pip install <package>` via subprocess
- Added `_run_shell(command, timeout)` — executes arbitrary shell commands
- Registered as actions `pip_install` and `run_shell`

### New: run and run_python actions
- **File**: `modules/file_manager_module.py`
- Added `_run(command, timeout)` — execute shell command
- Added `_run_python(code, timeout)` — execute Python code inline
- Registered as actions `run` and `run_python`

### New: Self-healing layer in orchestrator
- **File**: `modules/orchestrator_module.py`
- Added `_auto_heal()` — intercepts module errors in real-time:
  - **"X not installed"** → auto pip install X then retries original action
  - **"Unknown action: install_package"** → maps to `system_control.pip_install`
  - **"Unknown action: run_file"** → maps to `file_manager.run`
  - **Any unknown action** → tries `run_shell` as last resort
- Added `_record_failure()` — writes failures to `learning_engine.skill_memory` for offline analysis
- Added `_has_action_error()` / `_get_action_error()` helpers
- Updated system prompt to list new actions (`pip_install`, `run_shell`, `run`, `run_python`)
- Added critical rules for pip install and shell execution
- Wired `learning_engine` reference (set externally in `__init__.py`)

### Wiring: Learning engine into orchestrators
