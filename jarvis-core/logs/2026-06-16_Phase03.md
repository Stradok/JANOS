# Phase 3 — Memory Integration + Self-Healing — 2026-06-16

## Summary
Closed the feedback loop between episodic memory and agent decisions. Every major decision now queries episodic memory before executing. Built a self-healing critic that diagnoses failures and suggests fixes. Implemented routing engine that tracks agent/model success rates and feeds back into selection.

## What Was Built

### 1. RAG-Enhanced Reasoning Pipeline (`core/reasoning_pipeline.py`)
- `ReasoningPipeline.run(ctx)` — full 8-step pipeline:
  1. RAG recall from episodic memory (top-5 relevant episodes)
  2. Task analysis via model_router.classify_task()
  3. Model selection via model_router.select_for_task()
  4. Plan generation via PlannerAgent (with RAG context injection)
  5. Execution via ExecutorAgent
  6. Validation via CriticAgent with structured scoring
  7. Episode storage in episodic memory
  8. Score assignment based on errors
- `_format_rag_context()` — formats past episodes as annotated context (✅ success, ❌ failure)
- `CriticValidator` — validates output, calculates score, drives self-healing
  - `validate_output()` — checks for errors, empty output, scores -1.0 to 1.0
  - `self_heal()` — diagnoses failure root cause, proposes fix, queries episodic memory for similar past failures

### 2. Scoring → Routing Feedback Loop (`core/routing.py`)
- `RoutingEngine` — dedicated SQLite DB tracking agent and model success rates
- `record_agent_result()` — stores per-agent success/failure/runtime/score
- `record_model_result()` — stores per-model per-task-type success rates
- `get_best_agent(task_type)` — returns highest-scoring agent
- `get_best_model(task_type)` — returns highest-scoring model for task type
- `get_agent_rankings()` / `get_model_rankings()` — ranked lists for dashboard

### 3. Wire-up in main.py
- Pipeline, critic, and routing engine initialized at startup
- All 6 agents wired into the pipeline
- RAG context passed from episodic memory → planner → executor → critic
