# JAN Status Report — 2026-04-20

## System Version: v2.0 (Autonomous Multi-Agent + Self-Learning)

### Summary
Complete architecture overhaul from single-shot orchestrator to **14-agent autonomous system** with agentic think→act→observe→decide loops. Added **self-learning engine** with RAG pipeline, skill memory, and overnight autonomous learning. Every agent now has memory recall, RAG knowledge injection, and skill-based prompt optimization.

### Architecture Upgrade: v1 → v2

| Feature | v1 (Apr 12) | v2 (Apr 20) |
|---------|-------------|-------------|
| Orchestration | Single LLM call → 1 tool → done | Multi-step agentic loops (up to 20 steps) |
| Agents | 0 (monolithic) | 14 specialized agents across 7 tiers |
| Routing | LLM-only (30% JSON failures) | Keyword-first + LLM fallback |
| Memory | Save-only | Recall before run + save after |
| Learning | None | RAG + skill memory + overnight sessions |
| Fallback | None | Agent fails → retry with research agent |
| Frontend | None | JARVIS-style dark chat UI |
| Context | None | RAG + skill tips + memory injected per run |

### Self-Learning Engine
- **RAG Pipeline**: Web search → read page → chunk (500 words, 50 overlap) → MD5 dedup → ChromaDB vectors + SQLite metadata
- **Skill Memory**: Records every tool call outcome (agent/tool/action/input/success/error) → builds "what works" tips injected into agent prompts
- **Knowledge Explorer**: Picks topics → searches web → reads pages → ingests into RAG store
- **Overnight Sessions**: Analyze past failures → explore new topics → extract user preferences → repeat on configurable schedule
- **Background Runner**: Auto-starts on boot, runs every 6 hours for 30-minute sessions (configurable)

### Key Fixes Since v1
1. **Agent prompts fixed** — Every agent prompt now includes exact tool input formats matching actual module APIs
2. **Memory integration** — Pre-run recall (knowledge + preferences + conversations) + post-run save
3. **RAG injection** — Knowledge from learning engine injected into every agent run
4. **Skill tips** — Learned patterns ("this works", "this fails") prepended to agent system prompts
5. **Skill recording** — Every tool call outcome logged for continuous improvement
6. **Orchestrator fallback** — If agent fails → automatic retry with research agent
7. **Keyword routing** — Priority-ordered keyword matching (zero latency), LLM fallback for ambiguous
8. **Chat agent max_steps** — Fixed from 1 (broken) to 5 (functional)
9. **Spotify search-first** — Agent always searches before playing, not just toggle play/pause
10. **Frontend** — JARVIS-style dark chat UI with agent sidebar and step viewer
